# 待填写组号83 - 作业报告

## 一、程序功能介绍

本项目是基于 Qt/C++ 开发的桌面端记账本系统，面向个人与多人协作记账场景。程序围绕“记录收支、管理账本、设置预算、统计分析、多人共享”展开，提供从日常记账到预算提醒、图表分析和在线邀请协作的一整套功能。

### 1. 用户登录与数据隔离

程序启动后进入登录/注册界面。用户登录成功后，系统会根据当前用户名建立独立的数据目录，账本列表、选中账本、记账记录、预算、分类标签、头像和个人资料都会随登录用户变化，避免不同用户之间数据串用。联网状态下，登录和注册优先通过服务器完成；离线时仍保留本地用户校验能力，提升可用性。

### 2. 账本管理

账本管理页面支持创建账本、切换当前账本、查看账本详情、修改账本信息、删除账本以及加入多人账本。除默认账本外，用户创建的新账本会直接通过服务器创建为可邀请的多人账本，避免本地账本和在线账本重名重复。账本详情页会展示账本名称、备注、成员信息以及该账本下的收入、支出、结余和明细列表。

### 3. 收支记录

用户可以选择账本，新增收入或支出记录。每条记录包含日期、类型、分类、金额、备注以及可选图片凭证。明细页支持按时间展示、按分类筛选、修改记录、删除记录和查看凭证图片。修改记录时可以同步修改日期、类型、分类、金额、备注和图片。

### 4. 分类标签管理

系统内置常用收入和支出分类，如工资、生活费、餐饮、购物、交通等。用户可以在标签管理中新增或删除自定义分类。若某个标签被删除，系统会检测已有记录中失效的分类，并将其自动调整为“其他”，保证数据展示稳定。

### 5. 预算管理与首页提醒

预算页面支持按月份和当前账本设置预算金额，程序会读取当月支出并计算预算使用进度。首页展示本月预算、剩余预算提示、本月收入、本月支出、本月结余和记账笔数。预算提示会根据剩余比例切换颜色和提示文案，帮助用户及时控制消费。

### 6. 统计分析

统计页面使用 Qt Charts 展示当前账本的收支分析，包括本月收入分类占比、本月支出分类占比和历史月度收支趋势。用户切换账本后，统计页面会自动读取当前账本数据并重新绘制图表。

### 7. 多人账本与联网功能

项目实现了基于 TCP 的客户端和服务器。服务器维护用户、账本、成员关系、邀请码和共享记录。用户创建账本后，服务器生成邀请码；其他用户输入邀请码即可加入该账本。服务器会保存账本成员列表，并向在线成员发送加入、离开和新增记录通知。客户端支持通过配置文件或启动参数连接远程服务器，从而实现跨设备使用。

## 二、项目各模块与类设计细节

### 1. 主程序与窗口模块

`main.cpp` 负责应用初始化、命令行参数解析、服务器启动模式判断、客户端连接和登录流程控制。程序支持仅启动服务器、启动本地服务器、连接指定远程服务器等模式。登录成功后，主窗口 `Widget` 会以当前用户身份创建。

`Widget` 是主界面类，负责侧边栏导航、首页汇总、用户信息显示、头像菜单和页面切换。它内部组合了 `Record`、`Budget`、`Manage` 和 `datachartwidget` 等页面对象。首页的数据通过读取当前用户、当前账本和当月记录动态计算。

### 2. 用户与数据路径模块

`LoginDialog` 负责用户登录和注册。联网时调用 `NetworkClient` 的注册与登录接口；未连接服务器时使用本地 `users.json` 作为备用登录方式。

`UserData` 是新增的统一数据路径工具类，负责根据当前登录用户生成安全目录名，并提供账本文件、选中账本文件、标签文件、记录文件、预算文件、图片目录、个人资料文件和头像文件路径。该模块将原先散落在项目根目录的业务数据改为按用户隔离的 `data/用户名/` 结构，是多用户本地数据隔离的核心。

### 3. 账本管理模块

`Manage` 负责账本列表展示、当前账本切换、账本创建、编辑、删除、详情查看以及多人账本加入。新建账本时，`Manage` 会调用 `NetworkClient::createBook` 直接在服务器创建账本；服务器返回账本 ID 和邀请码后，本地列表只保存这一份账本，避免出现本地和在线同名账本。右键菜单提供修改账本信息、删除账本和加入多人账本入口。

`CreateBookDialog` 和 `EditBookDialog` 分别用于新建账本和修改账本的输入表单。`BookDetail` 用于展示账本基础信息和明细统计。

### 4. 记账记录模块

`AddOne` 是新增收支记录对话框。它会读取当前用户的账本列表和分类标签，并将记录追加到当前用户对应账本的数据文件中。若用户选择图片，程序会复制图片到当前用户目录下的账本图片文件夹。

`Record` 是明细管理页面，负责加载当前账本记录、展示列表、删除记录、编辑记录、按分类筛选和查看图片。`EditRecordDialog` 是记录修改弹窗，用于修改记录字段和图片凭证。

### 5. 预算模块

`Budget` 负责按账本和月份保存预算、读取当月支出、计算剩余预算和进度条显示。预算文件通过 `UserData::budgetFile` 存放在当前用户目录中。该页面顶部展示当前账本和月份，结果区展示预算金额、已使用金额、剩余金额和进度条。

### 6. 图表统计模块

`datachartwidget` 使用 Qt Charts 生成图表，包括收入饼图、支出饼图和月度趋势折线图。它通过 `selected_book.json` 获取当前账本，并读取该账本记录文件进行聚合统计。图表使用统一色盘，保证页面风格一致。

### 7. 标签模块

`TagManagerDialog` 负责收入和支出标签的维护。标签存储在当前用户目录下的 `tags.json`。新增记录、编辑记录和明细筛选均复用该标签文件，从而保持分类体系一致。

### 8. 网络模块

`NetworkClient` 是单例 TCP 客户端，封装服务器连接、登录注册、创建账本、加入账本、获取账本、获取成员、同步数据、离开账本等请求。它通过 Qt 信号向界面层返回结果，降低 UI 与网络通信的耦合。

`AccountingServer` 是 TCP 服务端，维护用户表、账本表、成员列表和服务器数据文件。服务端支持用户注册登录、创建账本、邀请码加入账本、获取账本、获取成员、添加记录、同步记录和离开账本。服务器使用 JSON 作为消息格式，便于调试和扩展。

## 四、项目总结与反思

本项目完成了一个较完整的桌面端记账系统。开发过程中，我们从基础的本地记账功能开始，逐步加入账本管理、分类标签、预算提醒、统计图表、个人资料和多人账本功能。项目最终形成了较清晰的页面划分和模块结构，用户可以围绕不同账本完成日常收支记录和统计分析。

项目的一个重点是数据组织方式的改进。早期版本中，不同用户共用同一批本地文件，导致登录用户变化后账本和记录仍然混在一起。后期通过 `UserData` 工具类统一管理用户数据路径，把账本、记录、标签、预算和个人资料都隔离到当前用户目录中，使系统更符合多用户使用场景。

另一个重点是多人账本功能。最初的邀请功能只是单机模拟，邀请码只在本地生成，跨设备无法使用。后来加入了 `AccountingServer` 和 `NetworkClient`，将创建账本、邀请码、成员关系和加入账本流程交给服务器处理，使邀请加入账本可以通过网络正常工作。我们也修正了“本地账本转换为在线账本”导致同名账本重复的问题，改为新建账本时直接创建服务器账本。

在 UI 方面，项目采用 Qt Widgets 实现多页面切换，并通过样式表优化主页、预算页、按钮和卡片区域。后期对首页预算提示、预算页当前账本提示、账本管理菜单等细节进行了调整，使交互更加清晰。

项目仍有可以继续完善的地方。第一，服务器端目前使用 JSON 文件保存数据，适合课程项目和小规模演示，但如果用于真实多人协作，应改为数据库并加入更严格的并发控制。第二，密码目前仍是明文保存，后续应使用哈希加盐方式存储。第三，联网记录同步目前已经有接口基础，但本地记录与服务器记录之间还可以进一步做自动同步、冲突处理和离线缓存。第四，部分 UI 仍使用绝对坐标布局，后续可以改为更完整的布局管理器，以提升窗口缩放和不同分辨率下的适配能力。

总体来看，本项目覆盖了 C++ 面向对象设计、Qt 界面开发、文件读写、JSON 数据处理、图表展示、TCP 网络通信和模块化设计等课程知识点。通过持续迭代，我们对桌面应用的功能拆分、状态管理和用户体验有了更深入的理解。
